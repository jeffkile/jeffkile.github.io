Interstate: SIP Fuzzer


As a researcher at UC Irvine I built the initial version of a multi-threaded state-full fuzzer used to find vulnerabilities in SIP phones.

A resulting paper was published at Defcon 15 by Dr. Ian Harris.


"Fuzzing" is the act of inserting incorrect data into a message in the hopes of producing an error in the software receiving the message

"SIP" stands for Session Initiation Protocol and is an open source protocol widely used in Voice Over IP (VOIP) phones


Fuzzing SIP is very important because it allows researchers to find and fix vulnerabilities in the protocol and its implementation that could otherwise be exploited by nefarious parties trying to listen in to or otherwise manipulate a secure phone call.




Features:

  • Written in Python
  • Multi-Threaded (The system can both send and receive information simultaneously)
  • State-full (System has knowledge of which state the phone is in and can fuzz different types of messages accordingly)
